# Gram Connect Operational Runbook

## Purpose

This runbook defines the minimum sequence required to start, validate, and review the local Gram Connect stack.

## Backend Startup

```bash
cd /home/anurag-basistha/Projects/TODO/Gram-Connect/backend
python3 -m venv .venv
source .venv/bin/activate
python -m pip install --upgrade pip
python -m pip install torch torchvision --index-url https://download.pytorch.org/whl/cu124
python -m pip install -r requirements.txt
python -m pip install pytest
python generate_canonical_dataset.py
python -m pytest -q tests test_suite.py
python run_full_verification.py
python -m uvicorn api_server:app --host 127.0.0.1 --port 8011
```

## Frontend Startup

Run the frontend in a separate terminal:

```bash
cd /home/anurag-basistha/Projects/TODO/Gram-Connect/frontend
npm install
printf 'VITE_API_BASE_URL=http://127.0.0.1:8011\n' > .env
npm test -- --run
npm run typecheck
npm run lint
npm run build
npm run test:e2e
npm run dev
```

The frontend is expected to open on the URL printed by Vite, usually `http://127.0.0.1:5173`.

## Canonical Runtime State

- Canonical data is generated by `backend/generate_canonical_dataset.py` and stored in `data/`.
- Live application state is persisted in `backend/runtime_data/app_state.json`.
- `backend/run_full_verification.py` trains and persists `backend/runtime_data/canonical_model.pkl` and verifies:
  - dataset integrity
  - recommendation inference
  - schedule filtering
  - image analysis
  - audio transcription
- Interrupted training may be resumed from `backend/runtime_data/canonical_model.best.pkl` or `backend/runtime_data/canonical_model.progress.pkl`.

## Manual UI Review

1. Open the Vite application.
2. Sign in as coordinator using `coordinator@test.com` / `password`.
3. Confirm that the dashboard loads, assignment actions are available, recommendation generation completes, problem submission succeeds, and uploaded images produce visual tags.
4. Sign in as volunteer using `volunteer@test.com` / `password`.
5. Confirm that the volunteer dashboard loads assignments, proof submission succeeds, and the volunteer profile saves correctly.

## Optional Dataset Overrides

The canonical model path is fixed to `backend/runtime_data/canonical_model.pkl`.
Optional dataset overrides remain available for regeneration and controlled experiments:
`GRAM_CONNECT_PEOPLE_CSV`, `GRAM_CONNECT_PROPOSALS_CSV`, `GRAM_CONNECT_PAIRS_CSV`, `GRAM_CONNECT_VILLAGE_LOCATIONS_CSV`, and `GRAM_CONNECT_DISTANCE_CSV`.

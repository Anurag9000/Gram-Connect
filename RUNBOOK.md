## 3. Seeded Runtime Model
- Canonical data lives in `data/` and is generated by `backend/generate_canonical_dataset.py`.
- Live app state is persisted in `backend/runtime_data/app_state.json`.
- `backend/run_full_verification.py` trains and persists `backend/runtime_data/canonical_model.pkl` and verifies:
  - dataset integrity
  - recommendation inference
  - schedule filtering
  - image analysis
  - audio transcription
- Interrupted training can be resumed from `backend/runtime_data/canonical_model.best.pkl` or `backend/runtime_data/canonical_model.progress.pkl`.

The canonical model path is fixed to `backend/runtime_data/canonical_model.pkl`.
